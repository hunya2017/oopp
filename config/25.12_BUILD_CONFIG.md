# ImmortalWrt 25.12 编译配置文档

## 概述
本文档详细说明了 ImmortalWrt 25.12 版本的编译配置，包括源码仓库、编译参数、脚本文件和构建环境配置。

---

## 1. 源码配置

| 配置项 | 值 | 说明 |
|--------|-----|------|
| **REPO_URL** | `https://github.com/immortalwrt/immortalwrt` | ImmortalWrt 源码仓库 URL |
| **REPO_BRANCH** | `openwrt-25.12` | 编译使用的分支 |
| **REPO_TAG** | (可选) | 如果指定则检出该标签，默认为空 |

---

## 2. 编译环境配置

| 配置项 | 值 | 说明 |
|--------|-----|------|
| **Build Platform** | `Ubuntu-22.04` | CI 运行环境 |
| **Feeds Configuration** | `feeds.conf.default` | Feed 源配置文件路径 |
| **Timezone** | `Asia/Shanghai` | 时区设置 |

---

## 3. 编译文件路径

| 配置项 | 路径 | 说明 |
|--------|------|------|
| **CONFIG_FILE** | `config/immortalwrt.info` | 编译配置文件（.config） |

| **SETTINGS_PATCH** | `smpackage/settings.patch` | 默认设置补丁文件 |

## 5. 构建执行步骤

### 5.1 初始化阶段
1. **释放磁盘空间** - 清理 Docker 镜像和不必要的工具链
2. **初始化系统环境** - 安装编译所需的依赖包
3. **更新时区和编译时间** - 设置时区为 `Asia/Shanghai`

### 5.2 源码准备阶段
1. **下载源代码** - Clone ImmortalWrt 仓库并检出 `openwrt-25.12` 分支
2. **缓存处理** - 使用 `klever1988/cachewrtbuild@main` 加速编译过程
3. **执行自定义脚本 1** - 运行 `DIY_P1_SH` 脚本（在 feeds update 前）

### 5.3 软件包准备阶段
1. **更新源** - `./scripts/feeds update -a`
2. **安装源和修改默认设置** - `./scripts/feeds install -a`
3. **修改 default-settings** - 使用 patch 或 sed 方式修改默认设置文件
4. **执行自定义脚本 2** - 运行 `DIY_P2_SH` 脚本（在 feeds install 后）

### 5.4 编译阶段
1. **下载固件包** - `make defconfig && make download`
2. **编译固件** - `make -j$(nproc)` 或 `make -j1 V=s`（如果并行编译失败）
3. **输出编译信息** - 显示已选插件列表和磁盘使用情况

### 5.5 后处理阶段
1. **整理固件文件** - 组织 firmware、package、buildinfo 三个目录
2. **打包 IPK 文件** - 收集所有 `.ipk` 文件并压缩
3. **生成发布标签** - 为 GitHub Releases 生成版本标签
4. **自动发布** - 上传到 GitHub Releases

## 6. 默认设置修改说明

默认设置文件位置：`package/*/default-settings/files/99-default-settings`

修改方式有两种：

### 方式 1：使用 Patch 文件
- 使用 `smpackage/settings.patch` 进行 patch
- 失败时自动回滚到备份文件

### 方式 2：使用 Sed 命令
- 在 `exit 0` 前插入自定义设置
- 确保脚本有执行权限

修改内容可包括：
- IP地址配置
- DHCP 范围设置
- WiFi 配置
- 时区和语言设置
- 自定义服务启动

## 7. 编译触发条件

### 7.1 定时任务
- **Cron:** `0 18 * * 5`
- **时间:** 每周五北京时间凌晨 2 点（UTC+8）

### 7.2 手动触发
- **Watch** - Star 仓库时触发
- **Workflow Dispatch** - 手动运行，支持参数：
  - `ssh` - SSH 连接调试
  - `CACHE_BUILD` - 是否使用缓存加速
  - `UPLOAD_FIRMWARE` - 是否上传固件
  - `UPLOAD_BUILDINFO` - 是否上传配置文件
  - `UPLOAD_PACKAGE` - 是否上传插件
  - `UPLOAD_RELEASE` - 是否发布到 Releases

### 7.3 仓库推送事件
- **Repository Dispatch** - API 触发

## 8. 输出格式说明

### 8.1 固件文件目录 (`firmware/`)
- 文件格式：`20240312_1430__immortalwrt-*.bin`
- 包含类型：`.bin`, `.gz`, `.img`, `.zst` 等
- 重命名规则：`{DATE_SUFFIX}__文件名{扩展名}`

### 8.2 构建信息目录 (`buildinfo/`)
- `.config` 文件（编译配置）
- `feeds.conf.default` 文件
- `.buildinfo` 文件（构建元数据）
- `.manifest` 文件（软件包清单）

### 8.3 IPK 包概览 (`ipk_packages.tar.gz`)
- 压缩格式：`tar.gz`
- 命名规则：`{DATE_SUFFIX}__ipk_packages.tar.gz`
- 包含所有编译的 IPK 软件包文件

## 9. 编译参数详解

### 9.1 并行编译
```bash
make -j$(nproc)  # 使用全部 CPU 核心并行编译
make -j1 V=s     # 单线程编译，显示详细输出（失败时用）
```

### 9.2 环境变量
```bash
BOOTSTRAP_DOWNLOAD_CI_LLVM: "false"  # 禁用 Rust CI LLVM 下载（避免 404）
BOOTSTRAP_ON_FAIL: "1"               # 构建失败时继续
```

## 10. 文件完整性校验

编译完成后自动进行：

1. **MD5 校验** - 对所有生成的固件文件计算 MD5
2. **文件统计** - 统计固件、配置、软件包文件数量
3. **空间占用** - 显示各部分占用的磁盘空间

## 12. 常见编译错误处理

| 错误类型 | 可能原因 | 解决方案 |
|---------|--------|--------|
| 源码 Clone 失败 | 网络问题或仓库不可用 | 检查 `REPO_URL` 和 `REPO_BRANCH` |
| Feed 更新失败 | Feed 源不可用 | 检查 `feeds.conf.default` 配置 |
| 并行编译失败 | 内存不足或依赖问题 | 使用 `make -j1 V=s` 单线程编译 |
| 空间不足 | 编译产物过大 | 增加 GitHub Actions 运行器磁盘空间 |
| Patch 应用失败 | Patch 文件与源码不匹配 | 使用 sed 方式或手动修改 |

## 15. 快速参考

### 更改编译分支
编辑 `.github/workflows/25.12.yml` 中的 `REPO_BRANCH`:
```yaml
REPO_BRANCH: openwrt-25.12
```

```yaml
CONFIG_FILE: config/immortalwrt.info
```

在手动运行中取消选中 `CACHE_BUILD` 选项

### 仅编译不发布
在手动运行中取消选中 `UPLOAD_RELEASE` 选项
