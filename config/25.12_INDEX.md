## 📊 核心配置参数速查

### 源码配置
```yaml
# 源码仓库
repo_url: https://github.com/immortalwrt/immortalwrt
repo_branch: openwrt-25.12

# 编译环境
build_platform: Ubuntu-22.04
timezone: Asia/Shanghai
```

### 文件路径
```yaml
# 配置和脚本
config_file: config/immortalwrt.info
diy_script_1: smpackage/immortalwrt-all-1.sh
diy_script_2: smpackage/immortalwrt-all-2.sh
settings_patch: smpackage/settings.patch
```

### 编译参数
```yaml
# 固件命名
firmware_name: immortalwrt-all-25.12

# 编译并发
parallel_jobs: $(nproc)  # 使用全部 CPU 核心

# 缓存加速
cache_enabled: true
cache_type: ccache
```

### 上传和发布
```yaml
# 默认开启
upload_firmware: true      # 上传固件
upload_buildinfo: true     # 上传配置
upload_package: true       # 上传 IPK
upload_release: true       # 发布到 Release
```

## 🔄 典型工作流程

### 情景 1：修改编译配置

```
1. 本地编译生成新的 .config
2. 复制到 config/immortalwrt.info
3. git add && git commit && git push
4. ✅ 自动触发编译
5. 编译完成后查看产物
6. 如果成功，保持配置；失败则调调整
```

### 情景 2：添加自定义源

```
1. 编辑 smpackage/immortalwrt-all-1.sh
2. 添加: echo "src-git xxx ..." >> feeds.conf.default
3. git add && git commit && git push
4. ✅ 下次编译自动应用
```

### 情景 3：修改默认设置

```
方式 A（推荐）：使用 Patch 文件
1. 编辑 smpackage/settings.patch
2. 提交并推送
3. ✅ 自动应用修改

方式 B：直接在脚本中修改
1. 编辑 smpackage/immortalwrt-all-2.sh
2. 添加 uci 或 sed 命令
3. 提交并推送
4. ✅ 自动应用修改
```

### 情景 4：调试编译问题

```
1. 进入 GitHub Actions
2. 手动运行 workflow
3. 启用 ssh 参数
4. 连接到 Actions 环境调试
5. 根据错误修改配置
6. 重新推送并编译
```

## 🔍 文件说明详解

### `config/immortalwrt.info`
- **类型：** OpenWrt .config 配置文件
- **用途：** 定义编译的目标设备、包含的软件包等
- **格式：** `CONFIG_XXX=y/m/n` 形式
- **修改方法：** 
  1. 手动编辑
  2. 或从本地 `make menuconfig` 生成
  3. 或通过脚本生成

### `smpackage/immortalwrt-all-1.sh`
- **执行时机：** `feeds update` 之前
- **主要功能：**
  - 修改 feeds 源配置
  - 添加自定义 feed
  - 预处理源码
- **常见操作：** 修改 `feeds.conf.default`

### `smpackage/immortalwrt-all-2.sh`
- **执行时机：** `feeds install` 之后，编译之前
- **主要功能：**
  - 应用 .config 配置
  - 修改源码细节
  - 准备编译环境
- **常见操作：** `make defconfig`, 修改各种配置

### `smpackage/settings.patch`
- **类型：** Unified Diff 格式补丁
- **用途：** 修改 `99-default-settings` 脚本
- **目标文件：** `package/*/default-settings/files/99-default-settings`
- **常见修改：** IP 地址、密码、DHCP 范围等

### `.github/workflows/25.12.yml`
- **类型：** GitHub Actions Workflow
- **用途：** 定义自动化编译流程
- **触发条件：** 定时、手动、推送、Star
- **关键步骤：** 初始化 → 源码准备 → 编译 → 上传 → 发布

## 🐛 故障排除

### 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|--------|
| 编译失败 | .config配置错误 | 检查 `config/immortalwrt.info` |
| Patch 失败 | patch 与源码不符 | 重新生成 `settings.patch` |
| 脚本出错 | 脚本语法错误或权限问题 | 检查 `.sh` 文件格式和权限 |
| Feed 不存在 | 源URL 错误或已删除 | 检查 `feeds.conf.default` 配置 |
| 空间不足 | 编译产物过多 | 清理旧 Artifacts 或增加存储 |

### 调试步骤

1. **查看工作流日志** - GitHub Actions 查看详细错误
2. **检查配置文件** - 对照文档验证各配置项
3. **本地测试** - 在本地环境重现问题
4. **启用 SSH 调试** - 进入 Actions 环境交互调试
5. **查看文档** - [25.12_BUILD_CONFIG.md](25.12_BUILD_CONFIG.md)
