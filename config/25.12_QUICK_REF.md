## 🚀 常用操作

### 1️⃣ 修改编译.config 配置

**文件：** `config/immortalwrt.info`

```bash
# 编辑此文件来调整编译参数
# 例如：启用/禁用某个软件包、调整设备目标等
```

**快速编辑方法：**
1. 本地编译生成新的 `.config`
2. 复制到 `config/immortalwrt.info`
3. 提交到仓库
4. 下次编译自动应用

---

### 2️⃣ 修改默认设置

**方式 A：使用 Patch 文件（推荐）**

编辑 `smpackage/settings.patch`:
```diff
--- a/package/emortal/default-settings/files/99-default-settings
+++ b/package/emortal/default-settings/files/99-default-settings
@@ -10,0 +11,5 @@
+uci set network.lan.ipaddr='192.168.10.1'
+uci set network.lan.netmask='255.255.255.0'
+uci commit network
```

**方式 B：直接在脚本中修改（DIY_P2_SH）**

编辑 `smpackage/immortalwrt-all-2.sh`:
```bash
# 在脚本中添加
uci set network.lan.ipaddr='192.168.10.1'
uci commit network
```

## 🔧 Workflow 参数说明

### 在 GitHub 中手动运行编译

进入 **Actions** → **immortalwrt-r3s** → **Run workflow**

| 参数 | 含义 | 默认 |
|------|------|------|
| `ssh` | SSH 连接到 Actions（调试用） | False |
| `CACHE_BUILD` | 使用缓存加速 | True ✓ |
| `UPLOAD_FIRMWARE` | 上传固件 | True ✓ |
| `UPLOAD_BUILDINFO` | 上传配置文件 | True ✓ |
| `UPLOAD_PACKAGE` | 上传 IPK 包 | True ✓ |
| `UPLOAD_RELEASE` | 发布到 Releases | True ✓ |

## 🐛 常见问题速解

### Q: 编译失败，怎么调试？
**A:** 
1. 查看 GitHub Actions 运行日志
2. 对比 `25.12_BUILD_CONFIG.md` 检查配置
3. 启用 `ssh` 参数进入 Actions 环境调试
4. 查看编译错误日志中的具体错误信息

### Q: 如何修改 IP 地址/密码？
**A:**
1. 编辑 `smpackage/settings.patch`
2. 或在 `smpackage/immortalwrt-all-2.sh` 中添加 `uci` 命令
3. 提交后下次编译自动应用

### Q: 如何添加新的源/插件？
**A:**
1. 编辑 `smpackage/immortalwrt-all-1.sh` 添加新的 feed 源
2. 或编辑 `config/immortalwrt.info` 启用编译选项
3. 提交后下次编译自动应用

### Q: 编译太慢怎么办？
**A:**
1. 确保 `CACHE_BUILD` 已启用（缓存加速）
2. 检查 `.config` 是否启用了不需要的功能
3. 等待 Ubuntu 运行器资源释放

### Q: 如何只编译不发布 Release？
**A:**
1. 手动运行 workflow
2. 取消选中 `UPLOAD_RELEASE` 参数
3. 点击 **Run**

### Q: 编译产物在哪里下载？
**A:**
1. **GitHub Actions 构建产物：** Actions → 选择运行 → 下方 Artifacts
2. **GitHub Releases：** Releases 页面，按最新日期查找

## 📝 配置修改工作流示例

### 示例 1：启用 WiFi 6

编辑 `config/immortalwrt.info`:
```
# 添加或修改以下配置
CONFIG_PACKAGE_kmod-mt7921e=y
CONFIG_PACKAGE_luci-app-wifischedule=y
```

### 示例 2：修改 LAN IP 地址

编辑 `smpackage/settings.patch`:
```diff
+uci set network.lan.ipaddr='192.168.1.1'
+uci set network.lan.netmask='255.255.255.0'
```

### 示例 3：预配置 WiFi SSID 和密码

编辑 `smpackage/immortalwrt-all-2.sh`:
```bash
uci set wireless.@wifi-device[0].disabled='0'
uci set wireless.@wifi-iface[0].ssid='MyWiFi'
uci set wireless.@wifi-iface[0].key='password123'
uci commit wireless
```
